About Onyx

I started this journey because I found that existing security tools didn't offer the robust protection I required for my specific workflow. Every tool I encountered seemed to miss the mark, lacking the precision and reliability that I demand from software.

It became clear that if I wanted a tool that truly served my needs, I would have to build it from the ground up myself. I spent countless hours designing the logic, ensuring that every layer of the system was built with security and speed in mind.

That is when I decided to create Onyx. I chose the name because it represents a solid, secure foundation—the kind of base that I need for my code to function without compromising safety.

Onyx is more than just a toolkit; it is a manifestation of the standards I hold for my own work. By building it myself, I have full visibility and control over how the information is handled, ensuring nothing is left to chance.
